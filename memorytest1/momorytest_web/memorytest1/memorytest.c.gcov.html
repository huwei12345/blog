     777             : O_DIRECT:
     778             : 也就是说O_DIRECT只是不使用内核的高速缓存，直接读写磁盘，导致我们自己使用的缓存必须是
     779             : 磁盘扇区对齐的（512*x），而且读写write、read也得是对齐的，文件和设备的偏移量也是
     780             : 对齐的（读写对齐便可直接满足条件）
     781             : 只需要在O_DIRECT情况下，使用mmap申请空间，且读写字节数对齐
     782             : 而对非O_DIRECT情况下，可使用malloc，字节数不对齐也行，但本例子是对齐的
     783             : 
     784             : 线程思路:主线程获取随机数写前4k块，将随机数告知其他线程然后等待，其他线程根据
     785             : 线程数循环获取块来写，直到全部写完（写的部分是互斥的），唤醒主线程，主线程通知检验线程检验，

     787             : 若写完一次再校验，则等主线程收到signal且写线程全部停止后再唤醒校验线程
     788             : 校验线程校验完成，通知主线程，主线程进行下一次循环     
     789             : 问题：
     790             : 1.多线程下的初次检验可能不成功，因为多个线程写，在获取块后写可能先写后续位置，而此时关闭程序
